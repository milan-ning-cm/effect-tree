# 🌳 效果樹 Effect Tree

**從終端效果倒推必然前提的專案拆解系統**——單一 Python 檔、零依賴、Markdown 是唯一真相源、人拖圖 / AI agent 改檔雙向即時同步。

---

## 這是什麼

專案管理常淪為任務清單:做完一堆事,卻說不出「這件事為什麼存在」。效果樹把專案倒過來管:

1. **先立終端效果**(要讓什麼、變得怎麼樣)——不是「做完某個東西」,而是做完之後世界的狀態;
2. **往回按必然邏輯推**:「要達成它,前面必須先達成什麼效果、必須做什麼」;
3. 樹上每個節點都答得出「**我支撐上面哪一層**」——答不出的,就不該做。

節點分三型:🔵 **效果**(世界的狀態,帶「判準」=看到什麼算達成)、🟠 **執行**(動作,帶「產出」落點)、🟢 **Context**(背景資訊,帶「來源」,不參與進度推導)。狀態五段:待動/進行中/待驗收/完成/擱置。系統會自動把「**▶下一步**」標在可動工的前沿節點上——它是從樹推導出來的,不是要人維護的欄位,所以永遠不會跟實況脫節。

## 60 秒快速開始

```bash
git clone https://github.com/milan-ning-cm/effect-tree.git
cd effect-tree
py tree.py        # Windows;其他環境用 python tree.py / python3 tree.py
```

瀏覽器自動開 `http://127.0.0.1:8778`,內建一個範例專案。要開自己的專案:左上下拉選單 →「**＋ 新增專案樹…**」輸入名稱即可。Windows 想要開機自動常駐+無視窗啟動,把 `launcher.vbs`(帶 `silent` 參數)捷徑放進「啟動」資料夾。

## 怎麼運作(架構一頁懂)

```
projects/<專案名>/_效果樹.md   ← 唯一真相源(Markdown,一行一節點)
        ↑↓ 逐區精準讀寫
tree.py(stdlib 零依賴 server) ← agent 用檔案或 API 讀寫
        ↑↓ HTTP
tree.html(SVG 可拖拉編輯器)   ← 人用瀏覽器操作,每 2.5 秒自動同步
```

- **人和 AI 共編同一份檔**:你在圖上拖,agent 直接改 md——畫面 2.5 秒內自動刷新;兩邊同時改,寫入以內容雜湊 token 防撞(撞到擋下重載,不互相蓋)。
- **為多 agent 而生的容錯**:別的 agent 手寫出格式變體(狀態別名、重複 ID)會被自動歸一/自動改號,不鎖檔;真正無法解析的爛行才會亮紅色橫幅鎖定編輯,防止被靜默重寫丟失。
- **截圖也進版本控制**:貼上的圖存 `projects/<專案>/_效果樹_assets/`,md 用 `{圖: 檔名}` 引用,跟著 git 同步。

## md 格式(agent 與人都照這個寫)

```markdown
<!-- effect-tree:start -->
- E1[效果|進行中] 以教學方式帶動課程銷售 {判準: 首波成交 N 筆}
  - E2[效果|待動] 掌握知識框架與教學法 {判準: 課綱完整且通過 E3}
    - X1[執行|待動] 蒐集資料、框架與教學文 {產出: notes/01.md}
    - C1[Context|待動] 講師背景與過往教材 {來源: 訪談}
<!-- effect-tree:end -->
```

| 元素 | 規則 |
|---|---|
| 一行=一節點 | `- <ID>[<型別>\|<狀態>] <內容> {附註}`;換行用字面 `<br>`(編輯器 Shift+Enter 自動轉) |
| ID | `E#`=效果、`X#`=執行、`C#`=Context;穩定不重編;新 ID=該前綴最大值+1;撞號自動改號 |
| 狀態 | 待動/進行中/待驗收/完成/擱置(常見別名自動歸一) |
| 層級 | 縮排 2 格=下一層;**子=父的必然前提**(箭頭指向它支撐的上層) |
| 附註 | 效果=`{判準: …}`、執行=`{產出: …}`、Context=`{來源: …}`;寫不出先留 `_待補_`,不硬湊 |
| `{圖: a.png, b.png}` | 節點附圖(可多張),檔案在 `_效果樹_assets/` |

標記行之外的內容(frontmatter、備註)工具永不改動,可自由書寫。

## 編輯器操作總表

| 想做什麼 | 怎麼做 |
|---|---|
| 新增節點 | hover 卡片右側 →「＋境/＋執/＋效」;或 Ctrl+V 把剪貼簿文字/**截圖**直接貼成新卡 |
| 編輯 | 點卡片開面板(Enter=存,Shift+Enter=換行;面板可拖移、可縮放) |
| 改掛/排序 | 拖卡到目標**中間**=成為子節點;拖到**上/下緣**=插到它前/後;綠線+游標標籤即時寫明落點;拖近畫布邊界自動平移 |
| 快速切狀態 | 點卡片上的狀態膠囊 |
| 多選 | **雙擊空白處**進入框選模式 → 拖出選取框;頂端浮出工具列可**整組改型別/狀態**;Ctrl+C/X/V 複製剪下貼上(可跨專案,截圖自動搬家)、Delete 刪除、拖曳=整組移動、Ctrl+拖=複製到落點 |
| 摺疊 | 卡片 −/＋ 鈕收子樹(留殘影條,點擊展開);▾/▸ 收長內容成一行 |
| 調寬 | 拖卡片右緣;同層欄位自動對齊 |
| 圖片 | 面板開著 Ctrl+V=貼進該卡(可多張);點縮圖開燈箱(滾輪縮放/拖曳平移) |
| 回復 | Ctrl+Z / Ctrl+Y(50 步;偵測到別人改檔會清空歷史,防止蓋掉他人變更) |
| 視野 | 空白拖曳=平移、滾輪=縮放、「縮放適配」=看全樹;卡片內網址自動成可點連結 |

## 建樹方法論(五條規則)

1. **先往上問到頂**:接到任何任務先問「這是為了什麼」,問到再上去就超出範圍為止——那才是樹根(「做 web app」的樹根是「帶動銷售」)。
2. **往回推必然前提**:每個效果問「要達成它,邏輯上必須先有什麼」;答不出「不先有它、上層會怎樣」就不掛。
3. **re-parent 是一等操作**:發現節點掛錯層,直接拖去對的地方,不重建。
4. **只展開活躍前沿**:當前要動工的路徑才拆到可執行的葉,其他分支停在效果層。
5. **判準寫不出就留 `_待補_`**:不硬湊;效果鼓勵寫判準、執行鼓勵寫產出。

## 真實用例(這套系統實際被這樣用)

以下都是本工具開發團隊的真實日常,不是想像:

1. **專案開局拆解**——接到「幫講師做教學 web app」,先推到「帶動課程銷售」再倒推,第一天就發現「符合講師本人風格」是必然前提,避免做出講師不能用的東西(見內建範例樹)。
2. **人 × 多 AI agent 的共同任務板**——多個 AI agent(跨機器、跨對話)並行推進同一專案:每個 agent 開工先讀樹、收工回寫狀態;人只看圖、拖拉調整、把「待驗收」驗成「完成」。撞寫防護與撞號自癒就是為此設計的,實測兩台機器的 agent 同時共編數週。
3. **對話即維護,樹自己長**——掛上 stop-hook(見 `hooks/`)後,你跟 agent 對話做事,對話結束 agent 自動把當天的進展同步上樹:狀態推進、新節點掛上、佔位樹根補填。人不用記得維護。
4. **自動日報/週報骨架**——對 `_效果樹.md` 做每日 git diff,「今天哪些節點推進了狀態」就是現成的日報骨架;比從聊天記錄和 commit 海裡撈重點準得多(我們的日報系統以樹 diff 當骨架、git 紀錄降為證據)。
5. **會議與腦暴收斂**——開會時把截圖、連結、想法 Ctrl+V 快速貼成卡(先不管結構),會後用框選+拖拉整組歸位;背景資訊收進 Context 卡,不污染進度視圖。
6. **驗收看板**——測完一批功能,框選整批卡、工具列一次切「完成」;「▶下一步」徽章隨時告訴團隊此刻最該動工的是哪幾張。
7. **知識/調研地圖**——調研型工作用 Context 卡掛來源連結與證據截圖,樹狀收斂「要回答終端問題,必須先弄清什麼」。

## FAQ

- **`py` 還是 `python`?** Windows 用 `py`,macOS/Linux 用 `python3`;launcher.vbs 內建用 `py`,環境不同請自行改。
- **port 被占?** `py tree.py --port 8899`。重複啟動安全(冪等:偵測到已在跑就直接開瀏覽器)。
- **專案想放別的資料夾?** `py tree.py --projects-dir <路徑>`;寫入白名單跟著錨到該目錄。
- **資料安全?** server 只綁 127.0.0.1;寫檔有路徑白名單;圖片上傳限 8MB、僅圖檔類型。
