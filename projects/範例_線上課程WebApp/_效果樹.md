# 範例_線上課程WebApp — 效果樹

> 這是內建範例:接到的任務是「做一個教學 web app」,但樹根不是「app 做完」——先往上問「這是為了什麼」,推到「帶動課程銷售」才開始往回拆。兩個標記行之間是工具管的區域;標記外自由書寫,工具不動。

<!-- effect-tree:start -->
- E1[效果|進行中] 讓講師能以 web app 上完一堂線上課程,並以教學方式帶動課程銷售 {判準: 首波課程成交 N 筆}
  - E2[效果|進行中] 掌握主題的知識框架與教學法,能把框架教給學生 {判準: 能產出完整課綱,且通過 E3}
    - C1[Context|待動] 講師背景:過往教材、既有 App 內容、學生輪廓 {來源: 講師訪談+公開內容}
    - E3[效果|待動] 教學框架符合講師本人的內容與風格 {判準: 對照講師過往教材無矛盾}
      - X1[執行|待動] 蒐集講師的過往教材與內容,整理其方法框架 {產出: notes/01_講師框架.md}
    - X2[執行|待動] 蒐集主題領域的資料、框架與教學文 {產出: _待補_}
  - E4[效果|待動] Web app 完成、可承載課程教學 {判準: 講師實際用它上完一堂課}
    - X3[執行|待動] 盤點 web app 技術選型與資料來源 {產出: _待補_}
<!-- effect-tree:end -->

## 備註
- 範例重點:①樹根是「銷售」不是「app 完成」(往上問到頂);②E3 一開始掛錯層被搬過(re-parent 是一等操作);③C1 是 Context 型=背景資訊,不參與「下一步」推導。
